## services/transaction_contents.proto
# Transaction Contents
The Signed Transaction message which forms the content of a transaction
`signedTransactionBytes`. This message is the result of several changes
to transaction message structure over time.

### Keywords
The key words &#34;MUST&#34;, &#34;MUST NOT&#34;, &#34;REQUIRED&#34;, &#34;SHALL&#34;, &#34;SHALL NOT&#34;,
&#34;SHOULD&#34;, &#34;SHOULD NOT&#34;, &#34;RECOMMENDED&#34;, &#34;MAY&#34;, and &#34;OPTIONAL&#34; in this
document are to be interpreted as described in
[RFC2119](https://www.ietf.org/rfc/rfc2119) and clarified in
[RFC8174](https://www.ietf.org/rfc/rfc8174).


<a name="proto-SignedTransaction"></a>

### SignedTransaction
A combination transaction bytes and a map of signatures.&lt;br/&gt;
This message contains a serialized `TransactionBody` in a byte array
and a `SignatureMap` that contains all of the signatures offered to
authenticate the transaction.

### Block Stream Effects
This content is recorded in the record stream exactly as received.


| Field | Type | Label | Description |
| ----- | ---- | ----- | ----------- |
| bodyBytes | [bytes](#bytes) |  | A byte array containing a serialized `TransactionBody`. &lt;p&gt; This content is what the signatures in `sigMap` MUST sign. |
| sigMap | [SignatureMap](#proto-SignatureMap) |  | A set of cryptographic signatures. &lt;p&gt; This set MUST contain all signatures required to authenticate and authorize the transaction.&lt;br/&gt; This set MAY contain additional signatures. |
