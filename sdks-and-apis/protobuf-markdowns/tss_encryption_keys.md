# Current and next TSS encryption keys

### Keywords
The key words &#34;MUST&#34;, &#34;MUST NOT&#34;, &#34;REQUIRED&#34;, &#34;SHALL&#34;, &#34;SHALL NOT&#34;,
&#34;SHOULD&#34;, &#34;SHOULD NOT&#34;, &#34;RECOMMENDED&#34;, &#34;MAY&#34;, and &#34;OPTIONAL&#34; in this
document are to be interpreted as described in
[RFC2119](https://www.ietf.org/rfc/rfc2119) and clarified in
[RFC8174](https://www.ietf.org/rfc/rfc8174).


<a name="com-hedera-hapi-node-state-tss-TssEncryptionKeys"></a>

### TssEncryptionKeys
A message containing a node&#39;s current and next TSS encryption keys, where
the next key (if present) will be switched to the node&#39;s current key during
the first transaction at the beginning of a staking period.&lt;br/&gt;


| Field | Type | Label | Description |
| ----- | ---- | ----- | ----------- |
| current_encryption_key | [bytes](#bytes) |  | If non-empty, a node&#39;s current TSS encryption key. |
| next_encryption_key | [bytes](#bytes) |  | If non-empty, the same node&#39;s next TSS encryption key. |
